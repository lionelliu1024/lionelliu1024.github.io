背景#

在修改hugo页面布局时，经常需要引入额外的js或css文件，刷新页面总是不满意，又不方便查看其中的环节及变量，遂想办法如何输出变量内容

官方指令#

debug.Dump#

Returns an object dump as a string.

Syntax

debug.Dump VALUE

Returns

    string

{{ $data := "" }}
{{ $p := "data/books.json" }}
{{ with resources.Get $p }}
  {{ $opts := dict "delimiter" "," }}
  {{ $data = . | transform.Unmarshal $opts }}
{{ else }}
  {{ errorf "Unable to get resource %q" $p }}
{{ end }}

<pre>{{ debug.Dump $data }}</pre>

[]interface {}{
  map[string]interface {}{
    "author": "Victor Hugo",
    "rating": 5.0,
    "title": "Les Misérables",
  },
  map[string]interface {}{
    "author": "Victor Hugo",
    "rating": 4.0,
    "title": "The Hunchback of Notre Dame",
  },
}

Output from this function may change from one release to the next.
Use for debugging only.

debug.Timer#

  Creates a named timer that reports elapsed time to the console.

Syntax

  debug.Timer NAME

Returns

  debug.Timer

Use the debug.Timer function to determine execution time for a block of code,
useful for finding performance bottle necks in templates.

The timer starts when you instantiate it, and stops when you call its Stop method.

{{ $t := debug.Timer "TestSqrt" }}
{{ range seq 2000 }}
  {{ $f := math.Sqrt . }}
{{ end }}
{{ $t.Stop }}

Use the –logLevel info command line flag when you build the site.

hugo --logLevel info

The results are displayed in the console at the end of the build. You can have as many timers as you want and if you don’t stop them, they will be stopped at the end of build.

  INFO  timer:  name TestSqrt total 12.429355ms
